## Опис проекту

**LimitWear** — веб-платформа для продажу лімітованих колекцій одягу з унікальним механізмом попередніх замовлень.

### Як це працює:
1. **Дизайнер** пропонує дизайн одягу на модерацію
2. **Адміністратор** перевіряє та публікує дроп з фіксованою кількістю та лімітом часу
3. **Покупець** робить передзамовлення — гроші **заморожуються** на балансі, але не списуються
4. Якщо всі місця заповнені до дедлайну — запускається **виробництво**, кошти списуються
5. Якщо дроп не набрав замовлень — всі кошти **автоматично повертаються**

### Ключові можливості:
- Система ролей: покупець / дизайнер / адміністратор
- Механізм заморозки коштів без реального списання
- Таймер зворотного відліку та прогрес-бар збору замовлень
- Вибір доставки через Нову Пошту (місто → відділення)
- Симуляція карткових платежів (поповнення / виведення балансу)
- Темна/світла тема та підтримка двох мов (UA / EN)
- Адмін-панель: модерація дизайнів, верифікація дизайнерів, управління дропами
- Автоматичний планувальник обробки завершених дропів

## Встановлення та запуск (Локально)

### Вимоги
- [Node.js](https://nodejs.org/) v18 або новіший
- [MongoDB](https://www.mongodb.com/try/download/community) (локально) або MongoDB Atlas

### 1. Клонування репозиторію

```bash
git clone https://github.com/wertikmarsik/bachelor-strubytskiy.git
cd bachelor-strubytskiy
```

### 2. Налаштування бекенду

```bash
cd server
npm install
```

Створіть файл `server/.env`:

```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/limitwear
JWT_SECRET=your_secret_key_here
JWT_EXPIRES_IN=7d
```

Запустіть MongoDB (якщо локально):
```bash
# Windows
net start MongoDB

# macOS/Linux
sudo systemctl start mongod
```

Заповніть базу тестовими даними:
```bash
npm run seed
```

Запустіть сервер:
```bash
npm run dev
```

### 3. Налаштування фронтенду

```bash
cd ../client
npm install
```

Створіть файл `client/.env`:

```env
VITE_API_URL=http://localhost:5000/api
```

Запустіть клієнт:
```bash
npm run dev
```

### 4. Відкрийте у браузері

```
http://localhost:5173
```
